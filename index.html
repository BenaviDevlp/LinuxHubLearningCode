Mis certificados

Crea tu primera página web pública con Linux

Objetivo: publicar esta misma plataforma (o cualquier carpeta con un index.html) en una dirección de internet para abrirla desde el celular, esté en la red que esté. Lo harás en 3 bloques: servidor local → acceso en tu Wi-Fi → acceso desde cualquier red. Todo con herramientas que ya trae Linux (Python y SSH), sin instalar nada extra.

Atajo: publícalo en 1 comando (20 minutos)

Si solo quieres compartirlo un rato, este proyecto incluye un script que levanta el servidor, abre el túnel público, te muestra la URL y se cierra solo a los 20 minutos:

# Publicar durante 20 minutos (puerto 8000 por defecto)
./publicar-20min.sh

# Otro puerto y otra duración (ej: puerto 8080, 30 minutos)
./publicar-20min.sh 8080 30

Copia la dirección https://....lhr.life que aparece y ábrela desde el celular. Para cerrar antes de tiempo, pulsa Ctrl + C. Si prefieres entender cada paso por separado, sigue el tutorial completo de abajo.

Antes de empezar

Necesitas tres cosas: una máquina con Linux, la carpeta de tu proyecto (la que contiene index.html) y conexión a internet. Abre una terminal con Ctrl + Alt + T. Comprueba que tienes Python 3 (viene de fábrica en Linux Mint):

python3 --version
# Debe mostrar algo como: Python 3.12.3

Paso 1 — Entra en la carpeta de tu proyecto

Sitúate en la carpeta donde está tu index.html. Sustituye la ruta por la tuya:

# Ir a la carpeta del proyecto
cd ~/Escritorio/folder

# Comprobar que aquí está tu index.html
ls

Si ves index.html en la lista, vas bien.

Paso 2 — Levanta el servidor local

Python puede servir una carpeta como página web con un solo comando. El 0.0.0.0 hace que sea accesible desde otros dispositivos de tu red, no solo desde esta PC:

# Arranca un servidor web en el puerto 8000
python3 -m http.server 8000 --bind 0.0.0.0

Déjalo abierto (mientras corra, el servidor está activo). Para detenerlo: Ctrl + C. Pruébalo en esta misma PC abriendo el navegador en http://localhost:8000/

Versión mejorada (opcional): en este proyecto incluimos un script servidor.py que además detecta tu IP automáticamente y evita la caché. Se ejecuta con python3 servidor.py o con ./iniciar-servidor.sh.

Paso 3 — Accede desde el celular en tu MISMA Wi-Fi

Averigua la IP local de tu PC (la dirección dentro de tu casa):

# Muestra tu IP local (toma la primera, tipo 192.168.x.x)
hostname -I | awk '{print $1}'
# Ejemplo de salida: 192.168.31.33

Con el celular conectado a la misma red Wi-Fi, abre en su navegador:

http://192.168.31.33:8000/
# (cambia la IP por la que te dio el comando anterior)

Paso 4 — Si no carga, abre el puerto en el firewall

Si el celular no conecta, suele ser el cortafuegos. Permite el puerto 8000:

sudo ufw allow 8000
# Para revisar el estado del firewall:
sudo ufw status

Hasta aquí solo funciona dentro de tu casa. Para entrar desde otra red (datos móviles u otra Wi-Fi), sigue con los pasos 5 y 6.

Paso 5 — Crea una clave SSH (solo la primera vez)

Para sacar tu servidor a internet usaremos un «túnel» por SSH a través del servicio gratuito localhost.run. Ese servicio te identifica con una clave SSH, así que primero la generamos (sin contraseña, pulsando Enter o con -N ""):

# Comprueba si ya tienes una clave
ls ~/.ssh/id_ed25519.pub 2>/dev/null

# Si no existe, créala (es instantáneo e inofensivo)
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""

Recuerda: la clave privada (id_ed25519) nunca se comparte; solo viaja la pública. Es exactamente lo que se explica en la lección de SSH del curso.

Paso 6 — Abre el túnel a internet

Con el servidor del Paso 2 todavía corriendo, abre otra terminal y ejecuta:

# Reenvía el puerto remoto 80 hacia tu servidor local (8000)
ssh -o StrictHostKeyChecking=accept-new -R 80:localhost:8000 localhost.run

Tras unos segundos verás una línea con tu dirección pública, parecida a:

... tunneled with tls termination, https://6d5ea71ee84b2d.lhr.life

Paso 7 — Ábrela desde cualquier red

Copia esa URL https://....lhr.life y ábrela en el celular usando datos móviles o cualquier otra Wi-Fi. ¡Ya no necesitas estar en tu red de casa! localhost.run también muestra un código QR en la terminal para escanearlo directo.

Cómo detener todo

Pulsa Ctrl + C en cada terminal: una para el túnel y otra para el servidor. Al cerrar el túnel, la URL pública deja de funcionar.

Errores comunes

  • «Permission denied (publickey)» al abrir el túnel: te falta la clave SSH. Haz el Paso 5.
  • «Address already in use»: el puerto 8000 está ocupado. Usa otro, por ejemplo 8080, en el servidor y en el túnel.
  • El celular no carga en tu Wi-Fi: revisa que sea la misma red y abre el puerto (Paso 4).
  • El comando se «cuelga» sin volver al prompt: es normal; el servidor y el túnel se quedan corriendo. No los cierres mientras los uses.

Tips del profesor

  • Necesitas dos terminales a la vez: una con el servidor (Paso 2) y otra con el túnel (Paso 6).
  • La URL gratuita cambia cada vez que reinicias el túnel. Para una fija, crea una cuenta gratuita en localhost.run y registra tu clave.
  • Esa URL es pública: cualquiera con el enlace puede entrar mientras el túnel esté abierto. Ciérralo cuando no lo uses.
  • Los datos de la plataforma se guardan en cada dispositivo (localStorage); el servidor solo entrega los archivos, no sincroniza tu progreso entre PC y celular.

Resumen rápido (los comandos clave)

# 1) Servidor local (terminal 1)
cd ~/Escritorio/folder
python3 -m http.server 8000 --bind 0.0.0.0

# 2) Tu IP local para la misma Wi-Fi
hostname -I | awk '{print $1}'      ->  http://TU_IP:8000/

# 3) Clave SSH (solo la primera vez)
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""

# 4) Túnel a internet (terminal 2)
ssh -o StrictHostKeyChecking=accept-new -R 80:localhost:8000 localhost.run
#    -> copia la URL https://....lhr.life y ábrela en el celular
